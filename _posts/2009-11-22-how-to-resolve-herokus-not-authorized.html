--- layout: post title: How to Resolve Heroku's 'not authorized to access newname' Git error. date: '2009-11-22T23:11:00.002-05:00' author: David Medinets tags: modified_time: '2009-11-22T23:13:53.145-05:00' blogger_id: tag:blogger.com,1999:blog-3207985.post-5565391816709747322 blogger_orig_url: http://affy.blogspot.com/2009/11/how-to-resolve-herokus-not-authorized.html --- When you see the 'not authorized to access newname' error, it probably. means that your Git configuration is incorrect. For example, my .git/config file had the following: 
[core]
 repositoryformatversion = 0
 filemode = true
 bare = false
 logallrefupdates = true
[remote "heroku"]
 url = git@heroku.com:newname.git
 fetch = +refs/heads/*:refs/remotes/heroku/*
Notice the highlighted words. Just change newname to the actual project name and you should be fixed.
