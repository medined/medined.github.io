--- layout: post title: Getting nova-manage to run in uksysadmin/OpenStackInstaller project. date: '2013-10-13T21:25:00.003-04:00' author: David Medinets categories: openstack modified_time: '2013-10-13T21:25:59.978-04:00' blogger_id: tag:blogger.com,1999:blog-3207985.post-2361176069733225150 blogger_orig_url: http://affy.blogspot.com/2013/10/getting-nova-manage-to-run-in.html year: 2013 --- While working to install OpenStack in a VirtualBox Ubuntu instance I ran into an issue running the 'nova-manage version' command. The error was of the form:

Failed to parse /etc/nova/nova.conf ... No ':' or '=' found in assignment

The resolution was fairly easy. Some of the parameters needed True or False values.

Apparently an older version of nova-manage used both name and value in the configruation. But a newer version wants just the name. (or vice versa - it's not clear to me).

I assigned True or False to each parameter that produced an error message. About five of them. I choose True or False at whim. When no more errors were reported, the program reported the following version:

2013.1.3
